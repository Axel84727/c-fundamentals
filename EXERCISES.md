## Tips:

- **No copies código de internet** - Escribe todo desde cero
- **Compila con warnings** - `gcc -Wall -Wextra -Werror`
- **Usa valgrind** - Para ejercicios con punteros/malloc
- **Comenta tu código** - Explica la lógica
- **Haz tests** - Al menos 3 casos de prueba por ejercicio
- **Refactoriza** - Si funciona pero está feo, mejóralo

---

## Recursos útiles:

- `man` pages en terminal
- cppreference.com (sección C)
- Compilador con `-Wall -Wextra` para encontrar errores
- gdb para debugging
- valgrind para memory leaks
